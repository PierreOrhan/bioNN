Simulator¶

Tools to run simulations in parallel on the CPU, for sparse matrix.

simulOfBioNN.simulNN.simulator.executeSimulation(name, inputsArray, initializationDic=None, leak=1e-13, endTime=20000)¶

Execute the simulation of the system saved under the name directory. InputsArray contain the values for the input species.

Parameters

  • name – directory path

  • inputsArray – The test concentrations, a t * n array where t is the number of test and n the number of node in the first layer.

  • initializationDic – can contain initialization values for some species. If none, or the species don’t appear in its key, then its value is set at leak.

Returns

The equilibrium values for everySpecies The nameDic to analyze them, useful to navigate in the created files for other plot. (But not necessary)

simulOfBioNN.simulNN.simulator.runForMultiprocess(X)¶

Solve mutiple time the ODE integration using odeint from scipy.

Parameters

X –

X = speciesArray,time,KarrayA,stochio,maskA,nameDic,myidx,output,outputPlot,outputDic,coLeak,maskComplementary With: speciesArray: 2d-array with each row as the initialization for one run of the integration

time: time step for output KarrayA: 3d-matrix with the reaction constant and stoichio coefficient maskA: mask nameDic: the dictionnary linking species name to their position in the array outputDic: name of species for which to compute the output coLeak: the small leak used for stability, added at the end of each computation of the derivative for every species maskComplementary: 1-mask

Returns

output: for each run (column)m, for each species in outputDic (row), the final value reached. outputPlot: for each each run (column)m, for each species in outputDic (row), all reached values.

Loader¶

In this file, we give a loader which generate the biochemical equations from the weights of a deep neural network. The weights should be stored in the format csv, with the following name:

weight_LayerNb

The weight dic should be a directory containing only weights files.

Bias should not be given in this file as we ignore it in the current algorithm.
